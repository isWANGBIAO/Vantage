# AI_Prediction 说明文档

## 功能概述
本模块用于分析日常生活记录，结合大模型能力，自动推断饮食与健康（如拉稀）之间的关系，辅助发现可能的过敏食物。支持数据清洗、分餐次结构化、健康事件结构化、统计分析、结果输出等全流程。

## 主要流程
1. **数据读取与清洗**
   - 支持从 Excel 或 CSV 文件读取原始生活记录。
   - 清洗无效数据：当天和次日健康状况都缺失的行会被剔除。
   - 只保留分析所需的关键字段，生成“有效数据表.csv”。

2. **健康标记与日期处理**
   - 自动判断“健康情况”字段是否为“拉稀”或“正常”。
   - 处理日期不连续的情况：只有当天和次日日期连续时，才将当天食物与次日健康情况关联。
   - 标记每一天的“拉稀”情况（只要当天或次日有拉稀就算1）。

3. **大模型结构化分析**
   - 利用阿里云大模型API（如qwen-max-latest），对“生活（饮食+社交+运动）”字段进行结构化，自动识别早餐、午餐、晚餐、小吃及其食物。
   - 对“健康情况”字段用大模型结构化提取所有拉稀事件，包括拉稀次数、严重程度（轻微/中等/严重）、发生时间（如早上/下午/晚上/凌晨）。
   - 生成“餐次-食物-拉稀事件”级别的明细表。
   - 支持断点续算：如果当天已生成分析表，则直接读取，无需重复调用大模型。

4. **统计分析与输出**
   - 统计每个餐次-食物的总出现次数、拉稀天数、拉稀总次数、严重/轻微天数、拉稀率。
   - 结果优先按“总出现次数”降序排序。
   - 输出明细表和统计表到csv文件。

## 主要输出文件
- `有效数据表.csv`：清洗后、仅保留关键字段的有效数据。
- `大模型分析表_YYYY-MM-DD.csv`：大模型分词和分类后的分析表，含所有分析相关字段。
- `餐次食物拉稀明细表.csv`：每条饮食-餐次-健康事件的结构化明细。
- `餐次食物拉稀统计表.csv`：每个餐次-食物的拉稀概率、次数、严重程度分布。
- `食物过敏分析表.csv`：传统分词统计结果，按出现次数和拉稀率排序。

## 依赖环境
- pandas
- numpy
- jieba
- openai（兼容阿里云API）
- tqdm
- openpyxl（如需读取Excel）

## 快速开始
1. 配置好阿里云API Key和Base URL环境变量。
2. 准备好原始生活记录数据（Excel或CSV）。
3. 运行 `analyzer.py`，自动完成数据清洗、结构化、统计和结果输出。

## 进阶说明
- 支持自动识别和结构化每条饮食记录的餐次（早餐、午餐、晚餐、小吃）及其食物。
- 支持自动结构化健康描述，提取所有拉稀事件（次数、严重程度、时间）。
- 可根据实际需求扩展结构化字段或统计逻辑。
- 支持断点续算，避免重复消耗大模型API。

---
如需自定义分析逻辑或字段，请修改 `analyzer.py` 中相关部分。
