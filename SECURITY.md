# Security Policy

## Reporting A Vulnerability

Please do not open a public issue for a security vulnerability or accidental
secret exposure. Report it privately to the repository owner through GitHub
security advisories or another private channel listed on the GitHub profile.

Include:

- affected version or commit
- reproduction steps
- expected impact
- whether any secret, screenshot, photo, health, finance, or location data may
  be involved

## Supported Versions

The default branch is the only supported development line until formal releases
are published.

## Sensitive Data Rules

Do not commit API keys, `.env` files, photos, screenshots, health exports,
finance workbooks, logs, local runtime data, or private prompt files.
